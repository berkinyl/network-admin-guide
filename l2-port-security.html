L2 – Port Security and Unauthorized Access

Unauthorized Network Access Due to Lack of Port Security

Layer 2 – Data Link Layer

Overview

This tutorial covers a common Layer 2 network security issue: unauthorized access to the local network due to the absence of port security configurations on switches. In environments where physical access to network ports is possible—such as open offices, classrooms, shared spaces, or public venues—malicious or unauthorized devices can connect to the network and gain access to internal resources without any authentication or restrictions.

Problem Scenario

In many switched networks, each switch port acts as a Layer 2 access point for a connected device. Without any restrictions, anyone with physical access to a wall port or switch can plug in their own device (e.g., a laptop or rogue access point) and instantly become part of the internal network.

Once connected, the unauthorized device may:

  • Access shared folders or internal services
  • Launch ARP spoofing or DHCP attacks
  • Attempt packet sniffing to collect sensitive information
  • Disrupt other users or perform denial-of-service attacks

This is a major security flaw, especially in corporate, educational, or governmental environments where physical access is not tightly controlled.

Symptoms and Risks

  • Unknown MAC addresses appear in the switch’s MAC address table
  • Network scanners detect unexpected devices on the subnet
  • End-users experience network slowdowns or strange behavior
  • Data leakage or unauthorized file access is reported
  • Logs show suspicious traffic or repeated port flapping

Technical Explanation

Switch ports in their default configuration accept frames from any connected MAC address. The switch dynamically learns the MAC address and stores it in its MAC address table, forwarding frames accordingly. Without control mechanisms, attackers can exploit this openness to impersonate legitimate devices or simply join the network unnoticed.

Step-by-Step Solution – Enabling Port Security

  • Limit the number of MAC addresses per port: Allow only one or a fixed number of MAC addresses per port.
  • Enable sticky MAC address learning: Automatically bind the first MAC address detected to the port.
  • Set violation actions:
    • protect: silently drop traffic
    • restrict: drop and log the violation
    • shutdown: disable the port completely (recommended)
  • Configure the interface (Cisco Example):

interface FastEthernet0/1
 switchport mode access
 switchport port-security
 switchport port-security maximum 1
 switchport port-security violation shutdown
 switchport port-security mac-address sticky
  • Monitor violations: Use show port-security and show port-security interface
  • Auto-recovery (optional):

errdisable recovery cause psecure-violation
errdisable recovery interval 300

Post-Fix Verification Steps

  • Connect an unauthorized device and confirm the port is disabled
  • Check logs and violation counters
  • Ensure legitimate devices continue to function
  • Review MAC address tables regularly

Best Practices

  • Enable port security on all access ports, not just critical ones
  • Use sticky MAC for a balance of control and automation
  • For dynamic environments, integrate with 802.1X
  • Regularly audit logs and alerts
  • Train staff to respond to violations quickly

Conclusion

Port security is a simple yet powerful tool for preventing unauthorized access at Layer 2. Without it, any open network jack becomes a potential entry point for attackers. Enforcing MAC address limitations and using proper violation responses significantly increases the security and integrity of the local network.
